Documenting Functions

Function Header Block

Information about a function or method is written in a header comment block that immediately precedes the function definition. The comment block should begin with a description of what the function does, followed by descriptions of the parameters and return value (if any). If your description is particularly long, you might want to consider using the @summary tag to provide a concise summary after the verbose description.

Global Function

/**
 * Adds the specified employee to the payroll system.
 * @param {Employee} emp - The employee to add to the system.
 */
function addEmployee(emp) {}

Class Method

/**
 * This method gets the full name of the employee. If the <var>includeMiddle</var>
 * parameter is set to <code>true</code>, the middle initial is included.
 * Otherwise, only the first and last name are included.
 *  
 * @summary Gets the full name of the employee.
 * @param {boolean} includeMiddle - Whether to include the middle initial.
 * @returns {string} Returns the full name of the employee.
 */
Employee.prototype.getFullName = function(includeMiddle) {};

Description

There are two ways to write the description of a function: you can write it before any other JSDoc tags, or you can use the @description tag to place the description anywhere in the comment block. In this guide, we write the description at the beginning of the comment block because it is cleaner and shorter.

Parameters

It is always a good idea to describe the purpose of each of the parameters and the type of value it is expecting. For this, we use the @param block tag. This tag uses the following syntax: 

@param [<expected type>] <parameter name> [-] <parameter description>

Note the optional hyphen (-) after the parameter name. This provides a visual separation between the parameter name and the parameter description to improve readability. The hyphen is not included in the HTML output. If the function takes more than one parameter, it helps to align the starting column of all of the descriptions.

Return Value

The @returns block tag indicates the type that is returned by the function and, optionally, a description of the value.

Reference Location

Global functions and class methods have their references in different parts of the documentation. Global functions are included with all other global objects on the Global page, while methods are included with the other members of the class that they belong to.

HTML Output

A function or method reference begins with the function signature and the return value if one was specified. This is followed by the brief summary (if any), and then the full description. The parameters are listed in a table similar to the one used for the class properties. Notice that when you set the type to a custom class, it creates a link to the class reference. The description of the return value and the return type are listed last.
