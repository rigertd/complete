Getting Started with JSDoc 3

by David Rigert

A Case for Documenting Your Code

The importance of code documentation is easy to overlook. Having well-written documentation not only helps other people understand the code you wrote, but also helps you understand what each part does and how it fits into the overall structure of the program. You probably have experienced a situation where your code made perfect sense when you were writing it, but when you go back and try to fix a bug several months (or weeks) later, you have no idea what the code is doing anymore. A small amount of effort documenting your code up front can pay big dividends down the road in terms of maintainability and code reuse.

Target Audience

This guide assumes you are familiar with JavaScript, HTML, and the Windows Command Prompt.

About JSDoc

JSDoc is a markup language used for documenting JavaScript code in multiline comment blocks. It uses a syntax similar to Javadoc, where annotation tags, or words preceded by an at-mark (@), are used to specify the documentation for specific parts of the code. A JSDoc comment starts with /** and ends with */.

You can also include HTML tags within JSDoc descriptions to create constructs such as bullet lists. Note that whitespace in description text is handled the same way as whitespace in HTML. If you want to have a bullet list, you must use the HTML <ul><li>â€¦</ul> notation.

Block Tags and Inline Tags

JSDoc has two types of tags: block tags that must be the first word to appear after the comment marker on any given line, and inline tags that appear in a description and are surrounded by braces ({}). Some tags take one or more arguments, each of which has a certain range. Some arguments must be a single word, while others continue until the next block tag or the end of the comment block. A list of all JSDoc block and inline tags is available on the @use JSDoc website. View the reference page for each tag to see syntax details. An optional argument is denoted by brackets ([]).

Unlike block tags, inline tags can appear in the middle of descriptions. JSDoc 3 includes only two inline tags: the {@link} tag for creating a hyperlink to other items in the documentation, and the {@tutorial} tag for creating a hyperlink to a tutorial created in a compatible format. Tutorials are beyond the scope of this guide. See the Tutorials section on the @use JSDoc website for more information.

Types

Some block tags optionally take a type as one of the arguments. The @param tag, for example, takes up to three arguments: the parameter name, the parameter type, and a description of the parameter. The type is surrounded by braces ({}) and can either be a native JavaScript type or a custom type. See the @type reference page for a complete list of the available type expression formats. Types are covered in greater detail in the Built-In and Custom Types section of this guide.

Required Software

The only software you need to get started writing comments with JSDoc is a text editor. If you want to provide HTML documentation for your code, however, you will need an API documentation generator. This guide uses the excellent JSDoc 3 toolkit with Node.js to generate the HTML examples. The easiest way to get started with JSDoc 3 is to install Node.js and use the NPM package manager. Download the installer for your operating system on the Node.js website.
