JSDoc Syntax

JSDoc Markup

JSDoc markup consists of a Javadoc-style comment block, block tags that begin with an at-mark (@), and inline tags surrounded by braces ({}). You can also include HTML tags within JSDoc descriptions to create constructs such as bulleted lists. Note that whitespace in description text is handled the same way as whitespace in HTML. If you want to have a bulleted list, you must use the HTML <ul><li>â€¦</ul> notation.

Block Tags

A block tag must be the first word to appear after the comment marker on any given line. Some tags take one or more arguments, each of which has a certain range. Some arguments must be a single word, while others continue until the next block tag or the end of the comment block. A list of all JSDoc block tags is available on the @use JSDoc website. View the reference page for each tag to see syntax details. An optional argument is denoted by brackets ([]).

Types

Some block tags optionally take a type as one of the arguments. The @param tag, for example, takes up to three arguments: the parameter name, the parameter type, and a description of the parameter. The type is surrounded by braces ({}) and can either be a native JavaScript type or a custom type. See the @type reference page for a complete list of the available type expression formats. Types are covered in greater detail in the Built-In and Custom Types section of this guide.

Inline Tags

Unlike block tags, inline tags can appear in the middle of descriptions. JSDoc 3 includes only two inline tags: the {@link} tag for creating a hyperlink to other items in the documentation, and the {@tutorial} tag for creating a hyperlink to a tutorial created in a compatible format. Tutorials are beyond the scope of this guide. See the Tutorials section on the @use JSDoc website for more information.
